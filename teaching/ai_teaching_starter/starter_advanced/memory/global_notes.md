# Global notes (advanced starter)

This file mirrors stable workflow preferences used in the main workspace.

## Brainstorming workflow

- Use markdown-first notes during pre-paper stages.
- Use numbered files in `notes/`:
  - `01_project_overview.md`
  - `02_literature_and_synthesis.md`
  - `03_model_notes.md`
  - `04_empirical_notes.md`
  - `05_research_plan.md`
- Keep `01` to `04` substantive and exploratory.
- Keep `05_research_plan.md` concise and decision-focused.

## File naming and ordering

- Use lowercase_with_underscores for folders/files.
- Keep governance filenames standardized (`README.md`, `STATUS.md`, `AGENTS.md`).
- Keep one canonical status file per tracked project when status tracking is used.

## Cleanliness

- Avoid loose backups in project roots.
- Store old versions in `old/` or dedicated backup folders.
- Keep build artifacts out of version control.

## Evidence discipline

- Do not invent references.
- Distinguish direct evidence from inference.
- Ask before assuming when design details are unclear.
